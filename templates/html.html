HTML

Basics

HTML isn't a programming language; it's a markup language used to tell your browser how to display the web pages you visit. It can be as complex or simple as the web designer desires. HTML consists of a series of elements that you use to nest or wrap different pieces of content to make the content appear or act in a certain way.
If we wanted a line to stand on its own, we could indicate that it was a paragraph by enclosing it in paragraph tags: < p > My cat is very irritated < /p >
Attributes contain additional information about an element that you don't want to appear in the actual content. In this case, class is the attribute name, and editor-note is the attribute value. The class allows you to give the element an identifying name that can later be used to access the element with style information and other information.
An attribute must always have: 1. A space between it and the element name (or the preceding attribute if the element already has one or more attributes). 2. The attribute name, followed by an equals sign. 3. The attribute value, enclosed in double quotation marks.

Anatomy of an HTML document

We've covered the basics of individual HTML elements, but they're not very useful on their own. Now we'll look at how individual elements are combined into a complete HTML page. Let's return to the code we wrote in our index.html (which we first encountered in the Working with Files article):
< !doctype html >
< html >
  < head >
    < meta charset="utf-8" / >
    < title >My first page< /title >
  < /head >
  < body >
    < img src="images/firefox-icon.png" alt="My test image" / >
  < /body >
< /html >
Here we have:

< !DOCTYPE html > — doctype. Back when HTML was young (around 1991/1992), 
doctypes were supposed to serve as a reference to a set of rules 
that an HTML page had to follow to be considered good HTML, 
which could mean automatic error checking and other useful things. 
However, nowadays, no one cares about this, 
and they're really just a historical artifact that must be included for everything to work properly. 
For now, that's all you need to know.

< html>< /html > — the < html > element. This element wraps all content on the entire page 
and is sometimes known as the root element.

< head >< /head > — the < head>  element. This element acts as a container for anything 
you want to include on an HTML page that isn't the content you're displaying to your page's users. 
These include things like keywords and page descriptions that will appear in search results, 
CSS styling for our content, encoding, and much more.
< body >< /body > — The < body > element. It contains all the content 
you want to display to users when they visit your page, 
whether it's text, 
images, videos, games, audio tracks, or anything else.
< meta charset="utf-8" > — This element sets the UTF-8 encoding for your document, 
which includes most characters from all known languages. 
Essentially, the document can now handle any text content you put in it. 
There's no reason not to set this, as it can help avoid some issues later.
< title >< /title > — The < title>  element. This element sets the title for your page, 
which is the name that appears 
in the browser tab when the page loads and is used to describe the page when you bookmark or favorite it.
